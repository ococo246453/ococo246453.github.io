[Hugo]架設自己的部落格 - Mac version

安裝需要的工具

  1. 安裝 Homebrew （這邊要注意Intel chip / Apple chip 安裝方式略有不同，本文將以Apple chip 做示範）

    • 首先開啟終端機，並執行 /bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"

      • 在終端機透過 echo $SHELL的顯示結果修改環境變數，以下圖為例，顯示結果為 /bin/zsh 。

      • 透過cd $home 回到根目錄，並執行nano .zshrc ，將
      echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

      加入，執行 control+x 提示是否修改Y 按下 Enter(return) 離開編輯器。

      • 執行source ~/.zshrc套用剛剛的修改，或者重啟終端機。
      • 開啟終端機，執行brew -v顯示安裝好的Homebrew版本，確認是否安裝成功。

2.安裝 Hugo

  • 在安裝前，先確認是否有安裝 Homebrew
    • 前往 Hugo 依照對應的作業系統教學指示安裝，本篇文主要是以MacOS為主，
    • 確認 Homebrew 有安裝成功後，執行brew install hugo等待安裝完成。

建立部落格

  1. 到想要的路徑執行hugo new site {名稱}
    • 執行完畢後，會在該路徑產出{名稱}的資料夾，內容如下圖

挑選主題

  1. 前往 Hugo Theme 挑選想要套用的主題
  2. 以筆者為例 選擇 Hugo Theme m10c 作為套用的主題，在選擇主題時，建議注意 Updated 日期，確認作者還有在持續維護，也可以透過GitHub Stars來作為考量依據，確認無誤後，點擊Download前往GitHub。

3.到剛剛執行 hugo new site{名稱}的路徑下的themes內，開啟終端機，並執行 git clone https://github.com/vaga/hugo-theme-m10c.git。

上傳GitHub

1.執行到這個部分，基本上雛型已經有了，現在需要到Github上建立兩個全新的Repository， 一個存放所有部落格資料使用，包含網站的Source code，相信大家不會希望自己網站的source code可以被大家任意瀏覽，所以記得把權限設為Private。

另外一個則是存放對外公開(Public)的頁面，而在這個對外公開的頁面Repository，有特定的命名規則 {id}.github.io {id} 的來源可以透過GitHub右上角的圖示查看，如下圖。

2.建立好之後，透過慣用的Git GUI 把兩個Repo都clone到本機端，並執行基礎的操作，如git init 這邊就不再贅述。

3.將剛剛透過hugo new site {名稱}建立好的資料夾，整個拉進存放所有，這個時候可以先git push一次到 remote repo 備份。

建立公開頁面

1.前往 hugo new site {名稱}的資料夾內，打開 config.toml，預設會看到如下圖。

2.進行設定檔的調整，將 baseURL 替換成自己的網址，以筆者為例， baseURL = "https://ococo246453.github.io"，並設定

themesDir = "../{名稱}/themes/"    #themes資料夾路徑
theme = "m10c"                    ＃主題名稱，已本篇文章的主題為例，為m10c

詳細更多的 config.toml設定，可以前往該主題作者的m10c GitHub查看 README.md

3.以上設定完畢後，可以在hugo new site {名稱} 開啟終端機，並執行hugo ，執行完畢後，在hugo new site {名稱}路徑下，會產出一個public資料。

透過終端機執行hugo server在本機端將網站執行起來，透過http://localhost:1313 可以查看結果，在終端機執行 contorl+c 可以停止本機網站。

4.確定網站在本機執行結果為預期結果後，將public資料夾的內容複製進Private權限的repo內，並推到remote repo上，執行成功後，可以在瀏覽器上前往 baseURL設定的網址查看，網站架設完畢。

新增文章

1.前往hugo new site {名稱}內的 content建立資料夾post並將.md格式的檔案放入，開啟終端機到hugo new site {名稱}執行 hugo，重新產出新的public資料夾，執行hugo server查看結果。

確定文章有上傳成功後，將public的內容複製到公開的repo推到remote，即可將文章顯示在網站上。

替換大頭貼

1.將圖片放入hugo new site {名稱}內的 static資料夾，開啟終端機到hugo new site {名稱}執行 hugo，重新產出新的public資料夾，執行hugo server查看結果。

注意事項

markdown文章的格式要注意兩點

  • 第一點，在文章的最上方，需要套用以下語法，內容可以替換成自己的。
---
title: Hugo_架設自己的部落格 - Mac version
tags: ["技能"]
categories: ["筆記"]
date: 2022-10-25T17:35:45
keywords:
- Hugo
description : "Hugo架站教學 架設自己的部落格 for MacOS"
---
  • 第二點，上方的格式內 date欄位的日期，不可晚於現實時間，否則會看不到文章，在現實時間超過設定的date後，文章才會被顯示，這邊的時間格式時區需要確認一下。
